局域网IP设置帮助

DHCP和TCP/IP的缺省值适用于大多数用户。

局域网TCP/IP设置

本页面为高级设置。网络中具有多台路由器时可由网络管理员设置。 设置更改后需要重启计算机方可令其生效。

  • IP地址:键入路由器的IP地址，其间以点号分隔（出厂缺省设定： 192.168.1.1).
  • IP子网掩码:子网掩码规定了IP地址中的网络编号部分。 路由器将根据您指定的IP地址自动计算出子网掩码。 除非需要划分子网，否则请使用255.255.255.0（路由器计算得出）作为子网掩码。
  • RIP方向:路由器可通过RIP（路由信息协议，RFC1058和RFC1389）与其它路由器交换路由信息。 通过选择RIP方向，可控制路由器收发RIP数据包的方式。 缺省为“无”。
    • 设置为双向或者仅外发时，路由器将定期广播其路由表。
    • 设置为双向或者仅内收时，路由器将对自身收到的路由表信息进行整合。
    • 设置为无时，路由器不会发送任何RIP数据包，并且会忽略自身收到的任何RIP数据包。 缺省为“无”。
  • RIP版本: 本选项控制的是路由器发送的RIP数据包的格式及广播方法。 （接收时路由器可识别两种格式。） 缺省情况下，本选项设置为“禁用”。
    • RIP-1具有通用支持。 除非网络设置比较特殊，否则RIP-1可能就足够用了。
    • RIP-2携带的信息要多一些。 RIP-2B和RIP-2M发送的路由数据均为RIP-2格式。
      • RIP-2B采用子网广播。
      • RIP-2M采用多播。 （参见下文说明。）

注意： 由于不属于路由器的设备不会侦听RIP多播地址，并且不会接收RIP数据包，因此多播可降低此类设备的负荷。 但是，只要有一台路由器使用了多播，那么用户网络上的所有路由器都必须使用多播。

将路由器用作 DHCP 服务器

本路由器在缺省情况下设置为一台DHCP（动态主机配置协议）服务器，从而为该路由器上连接的所有计算机提供TCP/IP配置。

除非ISP要求您更改这些设置，否则请保持选中将路由器用作DHCP服务器复选框。

如果ISP要求您清除这一复选框，那么您就必须在网络中另外配置一台DHCP服务器，否则就必须手工配置计算机。

  • 起始 IP 地址:本框规定的是IP地址池中的第一个连续地址。 缺省起始地址为192.168.1.2。
  • 结束 IP 地址:本框规定的是IP地址池中的最后一个连续地址。 缺省结束地址为192.168.1.254。

地址保留

如果为局域网中的某块PC网卡设置了一个IP保留地址，那么这块网卡每次访问DHCP服务器时，收到的总是同一个IP地址。 需要永久IP设置的服务器均应被分配IP保留地址。

要保留某个IP地址：

  1. 单击添加按钮。
  2. 从 地址保留表中选中待添加计算机旁边的单选按钮。
  3. 如果地址保留表中没有所需计算机，则请输入待添加计算机的IP地址、MAC地址和设备名称。
  4. 完成后单击添加按钮。

要编辑某个IP保留地址：

  1. 选中待编辑保留地址旁边的单选按钮。
  2. 单击编辑按钮。
  3. 编辑IP地址、MAC地址或者设备名称。
  4. 完成后单击接受按钮。

要删除某个IP保留地址：

  1. 选中待删除保留地址旁边的单选按钮。
  2. 单击编辑按钮。

保存或者取消更改

单击应用即可将新设置保存到路由器。
单击取消即可放弃所有尚未保存的更改。
